# 🚀 AI 1인 창업 마케팅 콘텐츠 자동화 서비스

본 프로젝트는 1인 창업자 및 소상공인이 블로그 포스팅 및 SNS 스레드 글을 7단계의 고도화된 마케팅 전략에 기반하여 자동으로 기획, 생성할 수 있게 돕는 **풀스택 웹 애플리케이션**입니다.

정적 랜딩페이지 디자인뿐만 아니라, **Google Gemini API** 연동을 통한 실시간 카피라이팅 및 이미지 자동 드로잉, 그리고 서버리스 환경에서도 영구적으로 사용 횟수를 트래킹할 수 있는 **Vercel KV(Redis)** 연동 시스템을 장착하고 있습니다.

---

## 🌟 핵심 기능

### 1. 7단계 스토리 마케팅 플래너 (gemini-3.5-flash)
단순한 텍스트 출력을 넘어, 실제 전문 카피라이터의 업무 순서를 엄격히 준수하여 콘텐츠를 점진적으로 설계합니다.
* **1단계**: 참고 링크 분석 및 비즈니스 맥락 추론
* **2단계**: 타깃 페르소나 및 핵심 고민, 전달 메시지 정의
* **3단계**: 키워드 기획 및 주제 선정
* **4단계**: 검색 상위 노출용 SEO 제목 기획 및 1개 선정
* **5단계**: 본문 글쓰기
* **6단계**: 매칭 이미지 4장 기획 (영문 프롬프트 구성)
* **7단계**: 연관 해시태그 10개 추출

### 2. 엄격한 텍스트 어조 및 구조 제어
* **구어체 존댓말**: 딱딱한 설명조가 아닌, 친근하고 부드러운 **`~더라고요 / ~거든요체`**로 작성됩니다.
* **상투어 전면 배제**: '오늘은 ~ 알아보겠습니다', '결론부터', '첫째/둘째', '따라서', '즉' 등 AI 특유의 인위적인 광고성 말투를 원천 금지합니다.
* **가독성 극대화**: 한 문단은 최대 3줄을 넘지 않도록 레이아웃이 줄바꿈 제어됩니다.
* **신뢰 결정타 문장 (대비 구조)**: 뻔한 영업 멘트를 배제하고, "나의 이익 A를 포기하더라도 고객의 이익 B를 중시한다"는 1인 사업가만의 솔직하고 의외인 사업 철학과 계기 문장을 본문에 강제 삽입합니다.

### 3. 고해상도(2K) 스마트폰 감성 AI 이미지 생성 (gemini-3.1-flash-image)
* **아이폰 스냅샷 느낌**: DSLR 스타일의 인위적인 스튜디오 샷 대신, **iPhone 16 Pro**로 촬영한 듯한 자연스러운 일상 조명, 살짝 불완전한 구도, 노이즈가 가미된 지극히 사실적인 질감으로 이미지를 생성합니다.
* **동아시아인/한국인 특화**: 인물이 묘사되는 경우 프롬프트 지침에 의거해 한국인의 외모, 헤어스타일, 따뜻한 감성 톤을 적용합니다.
* **초고해상도 2K 지정**: `responseModalities: ["IMAGE"]` 단독 호출 및 `"imageConfig": { "imageSize": "2K" }` 설정을 결합하여 무압축 초고화질 출력을 유도합니다.

### 4. Vercel KV(Redis) 기반 영구 사용 제한 트래킹
* 이메일 주소를 트래킹하여 **블로그 2회, 스레드 5회**의 무료 기회를 제공합니다.
* 한도 정보는 서버리스가 재부팅되어도 안전하게 영구 저장되는 클라우드 데이터베이스인 **Vercel KV (Redis)**에 기록됩니다.
* KV 관련 환경 변수가 연동되지 않은 로컬 실행 시에는 로컬 인메모리(임시 스토어) 시스템으로 하이브리드 자동 전환되므로, 어떠한 테스트 환경에서도 중단 없이 매끄럽게 가동됩니다.

## 🛠️ 기술 스택

* **Frontend**: HTML5, CSS3 (Vanilla CSS - Premium Dark UI Theme), JavaScript (Vanilla ES6+)
* **Backend**: Node.js, Express
* **Database**: Vercel KV (Cloud Redis) / Local In-Memory Fallback
* **AI API**: Google Gemini REST API (`gemini-3.5-flash` 및 `gemini-3.1-flash-image` 직접 REST POST 연동)

## 💻 로컬 실행 방법

1. **저장소 클론 및 패키지 설치**:
   ```bash
   npm install
   ```

2. **환경변수 설정**:
   프로젝트 루트에 `.env` 파일을 생성하고 발급받은 Gemini API 키를 등록합니다. (혹은 `.env.example` 파일을 복사하여 사용하세요.)
   ```env
   GEMINI_API_KEY=AIzaSyD...your_actual_api_key...
   PORT=8080
   ```
   * *참고*: 로컬 개발 시에는 Vercel KV 키를 비워두면 자동으로 로컬 인메모리 임시 카운트로 동작합니다.

3. **서버 구동**:
   ```bash
   npm start
   ```
   이후 브라우저에서 **[http://localhost:8080](http://localhost:8080)** 으로 접속합니다.

## ☁️ Vercel 클라우드 배포 및 영구 저장소(KV) 연동 가이드

본 프로젝트는 Vercel Serverless Function 배포 규격을 완벽하게 내장하고 있습니다. 10~15분 후 카운트가 날아가던 문제를 완벽하게 해결하기 위해 **Vercel KV** 데이터베이스를 연동하는 필수 단계입니다.

### 1단계: Vercel CLI로 프로덕션 배포 진행
```powershell
# 1. Vercel 로그인
powershell -ExecutionPolicy Bypass -Command "npx vercel login"

# 2. 실 프로덕션 배포
powershell -ExecutionPolicy Bypass -Command "npx vercel --prod --yes"
```
배포가 완료되면 콘솔 화면에 생성된 실서비스 프로덕션 접속 주소(예: `https://ai-content-generator.vercel.app`)가 출력됩니다.

### 2단계: Vercel 대시보드에서 `GEMINI_API_KEY` 환경변수 세팅
1. **[Vercel 대시보드](https://vercel.com/dashboard)**에 로그인합니다.
2. 배포된 프로젝트를 선택한 뒤 **`Settings`** $\rightarrow$ **`Environment Variables`** 로 이동합니다.
3. **Key**에 `GEMINI_API_KEY`, **Value**에 구글 Gemini API 실 키 값(`AIzaSy...`)을 입력하고 **`Save`**를 누릅니다.

### 3단계: Vercel KV (Redis) 생성 및 프로젝트 바인딩 (핵심)
이메일별 사용 횟수를 영구 보존하기 위해 Vercel KV 데이터베이스를 바인딩하는 단계입니다.
1. Vercel 프로젝트 대시보드 상단 탭에서 **`Storage`** 탭을 클릭합니다.
2. **`Connect Database`** 버튼을 누른 뒤, 데이터베이스 유형 중 **`KV (Redis)`**를 선택하고 **`Create`**를 누릅니다.
3. 데이터베이스 이름과 리전(지역)을 선택(기본값 유지)한 뒤 생성 단계를 완료합니다.
4. 생성이 끝나면 자동으로 배포 프로젝트에 연결(Link)되며, 프로젝트에 필요한 4가지 KV 환경 변수(`KV_URL`, `KV_REST_API_URL`, `KV_REST_API_TOKEN`, `KV_REST_API_READ_ONLY_TOKEN`)가 자동 등록됩니다.
5. 설정이 모두 완료되었으므로 **`Deployments`** 탭으로 이동하여, 가장 최근 배포의 우측 `...` 버튼을 누르고 **`Redeploy`** (재배포)를 클릭하여 환경변수를 함수에 완전히 반영합니다.

## 📋 프로덕션 영구 카운트 한도 검증 가이드 (20분 시간차 테스트 포함)

Vercel KV 연동 후, 서버리스 인스턴스가 슬립 모드로 진입했다 깨어난 이후에도 사용자 생성 횟수가 소멸하지 않고 안전하게 누적 유지되는지 확인하는 시나리오입니다.

### 1. 무료 한도 초과 및 잠금 작동 테스트
* [ ] **블로그(2회 제한) 잠금**: 동일 이메일 주소(예: `permanent-test@gmail.net`)로 블로그 글을 연속 2회 생성합니다. 3회째 생성 시도 시 모달 팝업 및 한도 초과 토스트가 정상 표출되는지 확인합니다.
* [ ] **스레드(5회 제한) 잠금**: 동일 이메일 주소(`permanent-test@gmail.net`)로 스레드 글을 5회 연속 생성합니다. 6회째 시도 시 결제 안내 팝업이 확실히 작동하는지 확인합니다.

### 2. 20분 시간차 영구 보존 테스트 (서버리스 슬립 극복 검증)
* [ ] **대기 시간 부여**: 생성 한도가 초과되어 차단된 이메일(`permanent-test@gmail.net`)의 상태 그대로, 브라우저 창을 닫고 **최소 20분 ~ 30분 동안** 사이트에 아무런 접속도 하지 않은 채 기다립니다. 
  - (이 대기 시간 동안 Vercel의 서버리스 람다 컨테이너가 유휴 상태로 완전히 내려가고 소멸합니다. 기존 인메모리 방식이라면 메모리가 공중분해 되어 카운트가 0으로 초기화되었던 타이밍입니다.)
* [ ] **슬립 해제 및 검증**: 20분 경과 후, 배포된 사이트에 다시 접속합니다. (첫 접속 시 람다가 깨어나느라 2~3초 로딩 지연이 발생할 수 있습니다.)
* [ ] **카운트 검증**: 이전과 동일한 이메일(`permanent-test@gmail.net`)을 적고 블로그나 스레드 생성을 다시 시도해 봅니다.
* [ ] **판정**: 기회가 복구되지 않고 **"무료 생성 횟수 한도가 초과되었습니다"** 토스트가 뜨며 결제 안내 모달창이 즉각 가로막는다면 **Vercel KV를 통한 실시간 영구 데이터 누적 기능이 완벽하게 가동**하는 것입니다!
